# Описание

Данная инструкция предназначена для развертывания персональной системы удаленного доступа на базе программного обеспечения `RustDesk`. В отличие от облачных решений (TeamViewer, AnyDesk), данная архитектура предполагает создание собственного сервера-посредника (Relay-сервера), что обеспечивает полный контроль над передаваемыми данными.

<br />

## 🧠 Цель проекта

Основная цель проекта - создание независимой, безопасной и высокопроизводительной инфраструктуры для удаленного управления компьютерными устройствами.

Задачи, которые решает система:

- Цифровой суверенитет: отказ от использования сторонних серверов, что исключает риск блокировок, принудительных обновлений или утечек через инфраструктуру третьих лиц.
- Стабильность соединения: минимизация задержек за счет использования собственного Relay-сервера.
- Централизация управления: возможность масштабирования сети (подключение неограниченного числа компьютеров к единому узлу) с унифицированными настройками доступа.
- Экономия: отсутствие лицензионных отчислений и лимитов на количество сессий.

<br />

## 🛡️ Рекомендации по безопасности

Безопасность системы целиком зависит от администратора. При эксплуатации сервера необходимо придерживаться следующих правил:

- Изоляция ключей: публичный ключ (`id_ed25519.pub`) предназначен для распространения клиентам. Приватный ключ (`id_ed25519`) должен храниться строго в закрытой директории сервера и не подлежит передаче третьим лицам.
- Сетевой периметр: настройте фаервол сервера (UFW/iptables) так, чтобы порты `21115–21119` были доступны только для необходимых IP-адресов (если работа происходит в рамках фиксированной офисной или домашней сети).
- Регулярное резервирование: бэкап папки пректа - единственный способ восстановить доступ к устройствам при выходе сервера из строя. Хранение архива с бэкапом в защищенном месте (например, в зашифрованном облаке или оффлайн-хранилище).
- Фиксация версий: использование конкретной версии образов в `docker-compose.yml` (например, `1.1.15`), чтобы избежать неожиданных сбоев после автоматических обновлений.

## 💡 Умения

Для успешного сопровождения проекта потребуются следующие навыки:

- Базовое администрирование Linux: понимание принципов работы терминала, умение редактировать файлы конфигурации (напр., `nano`), понимание прав доступа к файлам (`chmod`, `chown`).
- Docker-контейнеризация: умение запускать сервисы, проверять их состояние, анализировать логи (`docker compose logs`) и выполнять обновление контейнеров.
- Сетевая грамотность: понимание того, как работают порты, что такое UDP/TCP трафик и зачем нужна DNS-запись (А-запись для домена).
- Устранение неполадок: способность самостоятельно диагностировать проблему (например, "открыт ли порт?", "запущен ли сервис?", "верный ли ключ в конфиге?").
