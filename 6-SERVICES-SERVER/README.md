# Установка сервисов на сервер

В данном разделе описывается установка и базовая настройка сервисов, необходимых для работы серверной инфраструктуры. Сюда входят системные компоненты, панели управления и другие вспомогательные инструменты.

Эти сервисы используются как основа для развёртывания приложений, прокси-сервисов и дополнительных модулей. Рекомендуется выполнять все шаги последовательно, так как от корректной установки базовых компонентов зависит стабильность всей системы.

## 1. Установка Docker

`Docker` используется для запуска сервисов и панелей в изолированных контейнерах. Это позволяет удобно разворачивать приложения без зависимости от системных библиотек и конфигураций хоста.

Для установки выполните команду:

```bash
curl -fsSL https://get.docker.com | sudo sh
```

Устанавливаются следующие библиотеки:

- `curl` - утилита для отправки HTTP/HTTPS-запросов. Используется для загрузки установочных скриптов и взаимодействия с API через терминал
- `docker` - система контейнеризации, позволяющая запускать приложения в изолированных средах. Используется для управления контейнерами и их жизненным циклом

После установки необходимо добавить текущего пользователя в группу `docker` (замените `username` на фактическое имя пользователя):

```bash
sudo usermod -aG docker username
```

Чтобы изменения вступили в силу без перезагрузки системы:

```bash
newgrp docker
```

Включите `Docker` и добавьте его в автозапуск системы:

```bash
sudo systemctl enable --now docker
```

В редких случаях `Docker` может не запуститься с первого раза. В такой ситуации рекомендуется повторно выполнить установку и перезапустить сервис.
